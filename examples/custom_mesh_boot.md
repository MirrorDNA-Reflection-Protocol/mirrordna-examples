# Customizing Your MESH BOOT

## Overview

The default MESH BOOT from id.activemirror.ai works for most users. This guide covers advanced customization.

## MESH BOOT Structure

```markdown
# MESH BOOT — [Your Name]

## Identity
- Handle: ~your-handle
- Role: [Your role]

## Three Laws
1. TRUTH — Fact/Estimate/Unknown tagging
2. VAULT — Your files are canonical
3. MESH — Models coordinate as one

## Preferences
- Communication style: [terse/detailed]
- Code style: [language preferences]
- Output format: [markdown/plain]

## Context
- Current project: [project name]
- Working directory: [path]
- Key files: [list]
```

## Customization Options

### Communication Style

```markdown
## Communication
- Style: terse, no fluff
- Confirmation: "✓ Done" not "I have completed..."
- Questions: ask upfront, not mid-task
```

### Technical Preferences

```markdown
## Technical
- Languages: Python, TypeScript
- Framework: React, FastAPI
- Testing: pytest, always test first
```

### Domain Context

```markdown
## Domain
- Industry: [your field]
- Terminology: [domain terms]
- Constraints: [compliance, style guides]
```

## Tips

- Keep boots under 2000 tokens for compatibility
- Test in multiple models after customizing
- Version your boots (v1, v2, etc.)
